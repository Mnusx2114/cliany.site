# ADR-001: v0.2 迭代计划 — 工程质量 + 核心体验双轮驱动

**状态：** 已通过  
**日期：** 2026-03-29  
**决策人：** 韩孝冰  
**基线：** v0.1.1 (ee9bbf6)

## 背景

v0.1.1 已完成核心功能闭环（explore → codegen → replay），但存在以下瓶颈：
- 工程基础薄弱：无单元测试、无 CI、裸 except 静默吞错
- 用户体验断层：长流程无进度反馈、失败后无法恢复
- 扩展性受限：硬编码值、超大文件、仅支持本地 Chrome

## 决策

采用 **4 阶段渐进式迭代**，每阶段产出可独立发版的增量价值：

| 阶段 | 版本 | 主题 | 核心交付 |
|------|------|------|----------|
| Phase 1 | v0.2.0 | 稳固地基 | 异常治理 + 日志 + 配置中心 + pytest |
| Phase 2 | v0.3.0 | 体验升级 | 进度反馈 + 智能自愈 + CI/CD |
| Phase 3 | v0.4.0 | 场景拓展 | headless + 工作流编排 + 批量执行 |
| Phase 4 | v0.5.0 | 生态建设 | 适配器市场 + SDK + 安全加固 |

## 风险

- generator.py 拆分范围大，需充分测试回归
- headless 模式依赖 browser-use 上游支持程度
- 适配器市场需要设计 registry 协议，复杂度较高
